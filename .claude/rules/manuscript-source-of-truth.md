# Manuscript Source of Truth

**The Word `.docx` in `manuscript/` is the authoritative source for the paper.**

---

## Hierarchy

```
manuscript/china_innovation_tax.docx    ← AUTHORITATIVE (Word)
    ↑ tables copied from
output/tables/*.tex / *.xlsx / *.rtf    ← generated by Stata do-files
    ↑ figures copied from
output/figures/*                        ← generated by Stata / Python
    ↑ numbers derived from
data/final/*.dta                        ← analysis-ready datasets
```

**Direction of truth flows upward, never downward.**

- Never edit a number in the Word doc to "fix" a discrepancy — fix the Stata do-file
- Never paste figures from memory — regenerate from the do-file and copy the file

---

## Table Management

1. **Stata generates** tables to `output/tables/` (`.tex`, `.xlsx`, or `.rtf`)
2. **Author copies** into Word manually (or via reference)
3. **Never retype** numbers — always copy from the generated output
4. Each table in the manuscript should have a comment in the Stata do-file:
   ```stata
   // → output/tables/tab02_main.tex (Table 2 in manuscript)
   ```

### Table Formatting Standards (for `.tex` export)

- booktabs (`\toprule`, `\midrule`, `\bottomrule`) — no vertical lines
- Stars: `* p<0.10, ** p<0.05, *** p<0.01` in table note
- SE in parentheses immediately below coefficient
- N as integer with comma separator
- R² / Pseudo-R² to 3 decimal places

---

## Figure Management

1. **Stata or Python generates** figures to `output/figures/`
2. **Export format:** `.emf` or `.eps` for Word; `.pdf` for archival; `.tif` (300 DPI) for journal submission
3. **Never screenshot** a figure — always use the file from `output/figures/`
4. Each figure in the manuscript should have a comment in the generating code:
   ```stata
   // → output/figures/fig01_timeline.emf (Figure 1 in manuscript)
   ```

---

## Version Control for the Manuscript

- Commit `.docx` to git (Word documents are binary but git tracks them)
- Use commit messages that reference what changed: `"Update Table 2 — add industry×year FE column"`
- Do NOT commit to main with "WIP" manuscript — use a branch

---

## Slides (`slides/`)

- PowerPoint `.pptx` files are for conference/seminar presentations
- Slides derive from the manuscript, not the other way around
- Figures in slides should be the same files as in `output/figures/` — do not create slide-specific variants
- Slides are NOT the source of truth for any number — always verify against manuscript

---

## Consistency Checks (run before submission)

- [ ] Every number in the manuscript appears in a Stata output file
- [ ] Every table in the manuscript matches the most recent Stata-generated version
- [ ] Every figure in the manuscript matches the file in `output/figures/`
- [ ] Observation counts in text match table footnotes
- [ ] Sample period stated in text matches data construction do-file
