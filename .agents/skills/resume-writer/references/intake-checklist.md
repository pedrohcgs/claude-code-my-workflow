# Resume Intake Checklist

Use this checklist when the user's material is incomplete, scattered, or copied from chat notes.

## Blocking Questions

Ask only the questions needed to proceed:

1. Target language: Chinese, English, or bilingual?
2. Target role or industry: consulting, finance, data, product, research, general internship, or other?
3. Output format: `.tex + .pdf`, Word, Markdown, or review comments only?
4. Is one-page strict?

## Personal Information

- Full name as it should appear on the resume.
- Phone number.
- Email.
- Portfolio, LinkedIn, GitHub, personal website, or document link.
- City or location, only if relevant.

## Education

- University, school/college, degree, major.
- Start and expected graduation dates.
- GPA and scale.
- Rank or percentile if strong.
- Relevant coursework with scores if useful.
- Scholarships, awards, exchange programs, honors.

## Work Experience

For each role, collect:

- Organization, department/team, city, title, dates.
- 2-4 core workstreams.
- For each workstream: problem, your action, tools/methods, scope/scale, output, result, and insight.
- Any numbers: revenue, cost, sample size, company count, users, SKUs, pages, models, meetings, interviews, conversion, ranking, accuracy, time saved.
- What you personally owned versus what the team owned.
- Confidentiality constraints and anonymized client names.

## Projects

For each project, collect:

- Project/competition/research name, date, role, topic, award/result.
- Data source and sample size.
- Framework, model, software, or method used.
- Output format: report, deck, code, model, prototype, paper.
- Measurable result and original conclusion.

## Skills

- Languages and standardized scores.
- Programming, data, design, AI, finance terminals, office tools.
- Domain knowledge relevant to the target role.
- Selective campus or personal interests that signal real commitment.

## Rewrite Prompts

Use these prompts internally when transforming rough notes:

- What decision or stakeholder did this work serve?
- What was the hardest part of the work?
- What did the user do personally?
- What number proves scope or difficulty?
- What conclusion did the work support?
- Would this bullet still be impressive if the employer brand were removed?
